NOTES:
Validation tool has two modes: lock and project mode. In lock mode the validation tool only generates sboms for repos with a lock file
and in project mode validation tool only generates sboms for repos with project files. However, a good numebr of files have both lock 
and project files, so in the scenario that a repo has both, the validation tool will generate the same sbom for the repo because syft 
scans the entire repo. The mode parameter just determines which repos get scanned and which dont get scanned. Somewhat redundant. 

Validation tool can possibly change the repo from its intitial state so determining the best way to analyze all the repos is top 
priority as it will save time.
